Disclaimer - these patterns demonstrate what can be achieved by leveraging browser User Agent styles, but they use deprecate elements and attributes and should NOT be used

Disable ALL styles and this page will still retain it's essential appearance. It feels a shame not to be able to make use of this, but it's assumed that browsers NOT supporting CSS or uses browsing the web with styles deliberately disabled are so rare that it's not worth using deprecated markup for.

The alternate approach (Tiny FallBack Styles with Attributes - TFS+A) covers failed CSS scenarios, which is touch more likely (I assume).

Another Sample Card - this time with content

There is a source order problem here. Cards usually have their images above the heading, like it is here, but if the image is not presentational, it won't be accessible as part of the content the sits under the heading.

Currently the only way around this would be to include the image somewhere below the heading and use CSS to change the visual order, but in the Failed/No CSS scenario that's not possible.
